如果“浏览器先于网页语言”？

严格说，这不太符合 Web 的实际发展，但如果真先有浏览器而没有语言，它能做的只有两类事： 内置固定格式：浏览器把内容当纯文本或固定结构显示（几乎不可扩展）。 同时定义最小语法：现实中确实是这样发生的——第一代浏览器顺带就把“最小 HTML 语法”定了出来，因此语言与浏览器是“共同演化”的。

如果“网页语言先于浏览器”？

也能成立，但用途会变成：

作为标记/出版语言：像 SGML、TeX、troff 这类在 Web 前就存在的标记语言，用来排版打印或导出到其他格式（如 DVI、PostScript、PDF），用各自的查看器而不是“浏览器”来预览。 写好再“转换/渲染”：先有语言规范，再写一个渲染器（相当于浏览器的雏形）把它显示出来；没有浏览器时，就通过转换工具把标记文本转成可查看的格式。

早期网页到底是怎么做、怎么看的？

制作：用任何文本编辑器写 .html（完全手写）。 预览：用浏览器打开本地文件，如 file:///Users/me/test.html，改动后刷新。
